Alphabetic Order

Overview

In this variation of the last Job you will start to use processing components. In case you want an alphabetic order for the output and no more specific transformations you can use the tSortRow component. You will learn how to modify an existing Job starting again with duplicating it and redefine it afterwards. You also will pay attention to the error logs to understand how to correct your settings.

 

Duplicate and modify a Job

 

  1. Right-click on the UseMetadata Job and select Duplicate:

  1. Enter AscOrder for the new Job (like ascending order) and click OK:

  1. The Job is displayed and you want to insert the processing component tSortRow. One way to modify your Job, is to delete the link , to place the new component between input and output and link them again. So right-click the link and select Delete:

  1. Move the component tFileOutputXML to the right to have more space for the new component. You are looking for a "sorting" component. To find such components in the Palette you enter "sort" and click the magnifying glass agin and select tSortRow out of the results:

  1. Place the component tSortRow in the Workspace:

  1. You link the three components starting with a right-click on StatesCodes and select Row then click Main and connect to the component tSortRow:

  1. Go on with the next link and see the result:

 

Place tSortRow directly

 

Another way to enter the new component between the input and output component is to place it directly between the two components.

To prepare this option, delete the tSortRow component and connect the input and output again.

  1. Right-click on tSortRow and select Delete:

  1. Right-click on StatesCodes select Row and click Main and connect to tFileOutputXML:

  1. Select the link between the components and type tSortRow to place the new component. See also the description of the component which is displayed:

  1. The connections between the three components are set automatically:

  1. Save your Job.

 

 

Run the Job

 

  1. You do not see any warnings and decide to Run the Job. Select the Tab Run and click Run:

  1. You open the file out.XML in the folder C:\StudentFiles and see that the output is not ordered:

  1. To define an ascending order doubleclick the component tSortRow and click the green "+" at the bottom of the Criteria table:

  1. The field StateCode will be entered by the tool. Order is set to "asc" so the default order is ok. Lets assume you find this setting fine and you want to Run the Job again:

    .

  1. Select the Tab Run and Run the Job again. Then verify the error message. It indicates that column StateCode is set to "num" which is not correct. It has to be set to "alpha" to get an alphanumeric ordering:

  1. Double-click the tSortRow component again and select "alpha" for the column sort num or alpha?. Then Run the Job again:

  1. The Job runs without errors and displaying the file out.XML in the folder C:\StudentFiles, shows the correct output:
