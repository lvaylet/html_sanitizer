Create Metadata

Overview

In this lab you will learn to duplicate a Job and how to create and use the Metadata in the actual Job and how to reuse it in other Jobs.

Duplicating a Job shows how easy it is to reuse an existing Job and create a new one based on it.

You will create the Metadata first out of the duplicated Job. In the second part you will delete it and create the Metadata directly in the Repository using the Wizard without having a Job or component prepared.

Duplicate Job

 

  1. Right-click on "FirstJob 0.1" and click Duplicate:

  1. Enter Create_Metadata for the Input new name and click "OK":

  1. Double-click the new Job Create_Metadata and close the FirstJob:

  1. Double-click the component tFileInputDelimited named States to open the Tab Component and click the disc in order to save this schema in the Metadata to be reused further on:

  1. The Wizard for creating this type of Metadata will pop-up and you enter the "States" for the Name and enter text for the Purpose and Description.Then you click Next:

  1. The Wizard displays the next step and entries. File allows you to select another file for creating this Metadata. You will leave it to the "States.csv" file. The File Viewer shows a preview of the data. Click Next:

  1. Step 3 of the Wizard offers several options like Field Separator, Header, Footer, several CSV options and a Preview. We do not change the settings and click Next:

  1. Enter "StateCodes" for the Nameand enter text into the field Comment. Change the names for the columns in the table to "StateCode" and "StateName" and set the Length to "2" and "20" then click Finish:

  1. In the Repository expand the folder Metadata and display the new StatesCodes metadata:

    Note: It can now be reused in all your Jobs.

 

Delete and Create Metadata in the Repository

 

 

Overview

You created the Metadata for the component tFileInputDelimited. You used the wizard for creating the Metadata. There are several other wizards available for standard formats.

In case you cannot use the wizards you will create a Generic Schema by entering all the data manually.

Delete Metadata

 

  1. Expand the Metadata in the repository and right-click on States0.1 then click Delete:

  1. The Metadata will be created again using the same name. It is still in the Recycle bin and needs to be removed, else the system will retrieve the name. Right-click on Recycle bin and click Empty recycle binand answer with Yes:

  1. Right-click File delimited and click Create file delimited:

  1. The Wizard for creating this type of Metadata will pop-up and you enter "StatesCodes" for the Name and enter your text for the Purpose and Description:

  1. The Wizard displays the next step and entries. Server indicates that you only have a local connection for this installation and do not use remote data. File allows you to select another file for creating this Metadata. You will leave it set to the "States.csv" file. The File Viewer shows a preview of the data. Click Next:

  1. In step 3 of the Wizard set the Field Separator to Comma and leave Header, Footer empty. The CSV options will not be used and the Previewshows the correct data. Click Next:

  1. Enter "StatesCodes" for the Nameand enter text into the field Comment. Change the names for the columns in the table to "StateCode" and "StateName" and set the Length to "2" and "20" then click Finish:

  1. In the Repository expand the folder Metadata and display the new StatesCodes Metadata:
