Lab1 Getting Started

Overview

Usually when you are developing a Talend Data Integration project, you are not working in the final production environment. Because of this, some of the configuration parameters may be different in your development or testing environment than in production, such as the location of files in the directory structure. Additionally, you can handle errors in a development or test environment simply by observing the console, but when your project is deployed to production, you want it to deal with errors automatically.

In this lesson, you will create simple jobs , change parameter settings to get a first overview.

Objectives

After completing this lesson, you will be able to:

  • Change the settings in Talend software

  • Create new Jobs and duplicate Jobs

  • Use several components and create simple Jobs

  • Create metadata to reuse it in other Jobs

  • Create database connections and save as metadata

  • Use the component tMap to transform data
